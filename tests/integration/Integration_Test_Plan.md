## 1. 测试目标

本集成测试计划定义 ECH 水加热器 ECU 软件组件集成后的测试策略，验证：
- 各软件模块间接口正确性
- 模块集成功能符合软件需求 (SRSW)
- 集成后的性能满足系统需求 (SyRS)
- 故障注入和异常处理能力

## 3. 测试策略

### 3.1 集成方法

采用 **增量式集成** 策略，分阶段进行：

| 阶段 | 集成内容 | 测试重点 | 环境 |
|------|----------|----------|------|
| Phase 1 | 温度控制 + PWM 驱动 | 占空比输出正确性 | SIL |
| Phase 2 | 温度采集 + 温度控制 | 闭环控制逻辑 | SIL |
| Phase 3 | 状态机 + 上述模块 | 状态转换与使能控制 | SIL/HIL |
| Phase 4 | LIN 通信 + 状态机 | 命令解析与状态上报 | HIL |
| Phase 5 | 故障诊断 + 全部模块 | 故障注入与安全响应 | HIL |

### 3.2 测试类型

| 测试类型 | 描述 | 占比 |
|----------|------|------|
| 功能测试 | 验证集成功能符合需求 | 50% |
| 接口测试 | 验证模块间数据传递正确 | 20% |
| 异常测试 | 验证故障注入后的安全响应 | 20% |
| 性能测试 | 验证时序和响应时间 | 10% |

### 3.3 测试环境

| 环境 | 用途 | 配置 |
|------|------|------|
| SIL (Software-in-Loop) | Phase 1-3 | PC + GCC + CMocka 测试框架 |
| HIL (Hardware-in-Loop) | Phase 4-5 | ECU + dSPACE/NI 实时仿真器 + 负载模拟 |

### 3.4 通过标准

| 标准项 | 要求 |
|--------|------|
| 测试用例执行率 | 100% |
| 测试用例通过率 | ≥95% (P0 需求 100%) |
| 代码覆盖率 (集成层) | 语句覆盖 ≥90%, 分支覆盖 ≥85% |
| 严重缺陷数 | 0 (P0 需求相关) |
| 一般缺陷数 | ≤5 且有缓解措施 |

## 4. 集成测试场景

### 4.1 场景 1: 温度闭环控制集成

**场景 ID**: ITS-001  
**测试阶段**: Phase 1+2  
**涉及模块**: 温度控制、PWM 驱动、温度采集

**场景描述**:
验证温度控制模块根据目标温度和实际温度的差值，通过 PID 算法计算正确的 PWM 占空比输出。

**测试步骤**:
1. 初始化温度控制模块、PWM 驱动、温度采集模块
2. 设置目标温度为 50°C
3. 模拟实际温度从 25°C 开始上升
4. 监控 PWM 占空比输出变化
5. 当实际温度接近 50°C 时，验证占空比逐渐降低
6. 验证稳态时温度波动 ≤ ±2°C

**期望结果**:
- PWM 占空比随温度误差减小而平滑降低
- 无剧烈振荡或超调
- 稳态误差在 ±2°C 以内

**追溯需求**: SWE-F-001, SWE-F-004, SWE-F-007, SWE-F-008

---

### 4.2 场景 2: 状态机与使能控制集成

**场景 ID**: ITS-002  
**测试阶段**: Phase 3  
**涉及模块**: 状态机、温度控制、PWM 驱动

**场景描述**:
验证状态机根据使能请求正确转换状态，并控制 PWM 输出。

**测试步骤**:
1. 系统初始化，进入 STANDBY 状态
2. 验证 STANDBY 状态下 PWM 输出为 0%
3. 发送使能请求
4. 验证状态转换为 RUNNING
5. 验证 RUNNING 状态下 PWM 输出为 PID 计算值
6. 发送禁用请求
7. 验证状态转换回 STANDBY
8. 验证 PWM 输出恢复为 0%

**期望结果**:
- 状态转换正确
- PWM 输出与状态同步
- 状态转换响应时间 ≤ 50ms

**追溯需求**: SWE-F-005, SWE-F-006, SWE-F-014, SWE-F-015, SWE-F-016, SWE-F-017

---

### 4.3 场景 3: LIN 通信与状态上报集成

**场景 ID**: ITS-003  
**测试阶段**: Phase 4  
**涉及模块**: LIN 通信、状态机、温度采集

**场景描述**:
验证 LIN 通信模块正确解析 HVAC 请求，并周期性上报系统状态。

**测试步骤**:
1. 初始化 LIN 通信模块
2. 通过 LIN 发送目标温度请求 (40°C)
3. 通过 LIN 发送使能请求
4. 验证系统进入 RUNNING 状态
5. 监控 LIN 总线，验证状态帧每 100ms 发送
6. 验证状态帧中包含：实际温度、占空比、系统状态
7. 修改目标温度，验证系统响应

**期望结果**:
- LIN 请求解析正确
- 状态帧周期发送
- 数据内容准确

**追溯需求**: SWE-I-001, SWE-I-002, SWE-I-003, SWE-I-004, SWE-I-005, SWE-I-022, SWE-I-023

---

### 4.4 场景 4: 过温保护集成

**场景 ID**: ITS-004  
**测试阶段**: Phase 5  
**涉及模块**: 故障诊断、温度控制、PWM 驱动、状态机

**场景描述**:
验证当检测到过温条件时，系统正确触发保护并进入安全状态。

**测试步骤**:
1. 系统正常运行中 (目标 50°C，实际 45°C)
2. 通过 HIL 注入过温故障 (模拟温度 90°C)
3. 监控 PWM 输出切断响应时间
4. 验证状态转换为 FAULT
5. 验证 LIN 上报故障状态
6. 验证 DTC 记录过温故障
7. 尝试发送使能请求，验证系统拒绝

**期望结果**:
- PWM 在 10ms 内切断
- 状态转换为 FAULT
- DTC 正确记录
- 故障未清除前无法重启

**追溯需求**: SWE-S-001, SWE-S-002, SWE-S-003, SWE-S-013, SWE-S-014

---

### 4.5 场景 5: LIN 通信丢失保护

**场景 ID**: ITS-005  
**测试阶段**: Phase 5  
**涉及模块**: LIN 通信、状态机、故障诊断

**场景描述**:
验证当 LIN 通信丢失时，系统进入安全状态。

**测试步骤**:
1. 系统正常运行中
2. 通过 HIL 关闭 LIN 总线
3. 计时并监控系统状态
4. 500ms 后验证系统进入安全状态
5. 验证 PWM 输出为 0%
6. 恢复 LIN 通信
7. 发送使能请求，验证系统可恢复

**期望结果**:
- 通信丢失 500ms 后进入安全状态
- PWM 切断
- 通信恢复后可正常重启

**追溯需求**: SWE-S-006, SWE-S-007

---

### 4.6 场景 6: 软启动集成

**场景 ID**: ITS-006  
**测试阶段**: Phase 3  
**涉及模块**: 温度控制、PWM 驱动

**场景描述**:
验证系统启动时 PWM 占空比按限制速率爬升。

**测试步骤**:
1. 系统从 STANDBY 进入 RUNNING
2. 目标温度 50°C，实际温度 20°C (大温差)
3. 监控 PWM 占空比从 0% 开始的变化
4. 测量占空比爬升速率
5. 验证爬升速率 ≤ 10%/s

**期望结果**:
- 占空比平滑爬升
- 爬升速率 ≤ 10%/s
- 无阶跃变化

**追溯需求**: SWE-F-003

---

### 4.7 场景 7: 水泵联动控制

**场景 ID**: ITS-007  
**测试阶段**: Phase 3  
**涉及模块**: 状态机、水泵控制 (DIO)

**测试步骤**:
1. 系统处于 STANDBY 状态
2. 验证水泵继电器为 OFF
3. 发送使能请求，进入 RUNNING
4. 验证水泵继电器为 ON
5. 发送禁用请求，返回 STANDBY
6. 验证水泵继电器为 OFF

**期望结果**:
- 水泵与状态同步
- 响应时间 ≤ 50ms

**追溯需求**: SWE-F-013, SWE-F-014
