## 1. 引言

### 1.1 目的

本报告记录 ECH 水加热器 ECU 的危险分析与风险评估 (HARA) 结果，确定安全目标及其 ASIL 等级。

### 1.2 范围

本 HARA 覆盖 ECH ECU 在车辆运行中的所有可预见使用场景，包括：
- 正常驾驶工况
- 极端环境条件
- 误用场景
- 故障场景

### 1.3 定义与缩略语

| 术语 | 定义 |
|------|------|
| HARA | Hazard Analysis and Risk Assessment (危险分析与风险评估) |
| ASIL | Automotive Safety Integrity Level (汽车安全完整性等级) |
| HMI | Human Machine Interface (人机界面) |
| ECH | Electric Cabin Heater (电动座舱加热器) |

## 2. HARA 方法

### 2.1 风险评估参数

根据 ISO 26262-3，风险评估基于三个参数：

| 参数 | 等级 | 描述 |
|------|------|------|
| **S (Severity)** 严重度 | S0 | 无伤害 |
| | S1 | 轻度至中度伤害 |
| | S2 | 重度至危及生命的伤害 (可存活) |
| | S3 | 危及生命的伤害 (不可存活) 或 死亡 |
| **E (Exposure)** 暴露概率 | E0 | 不可信 |
| | E1 | 极低概率 (<1%) |
| | E2 | 低概率 (1-10%) |
| | E3 | 中等概率 (10-50%) |
| | E4 | 高概率 (>50%) |
| **C (Controllability)** 可控性 | C0 | 一般可控 (>99%) |
| | C1 | 简单可控 (90-99%) |
| | C2 | 正常可控 (50-90%) |
| | C3 | 难以可控 (<50%) |

### 2.2 ASIL 等级确定矩阵

| S \ E,C | E1,C1 | E1,C2 | E1,C3 | E2,C1 | E2,C2 | E2,C3 | E3,C1 | E3,C2 | E3,C3 | E4,C1 | E4,C2 | E4,C3 |
|---------|-------|-------|-------|-------|-------|-------|-------|-------|-------|-------|-------|-------|
| **S1** | QM | QM | QM | QM | QM | QM | QM | QM | A | QM | A | A |
| **S2** | QM | QM | A | QM | A | A | QM | A | B | A | B | C |
| **S3** | A | A | B | A | B | C | A | B | C | B | C | D |

> QM = Quality Management (无需 ASIL 等级)

## 9. ASIL 分解

### 9.1 分解原则

根据 ISO 26262-9，ASIL 可在满足以下条件时分解：
- 充分独立性
- 无共因故障
- 无级联故障

### 9.2 拟议分解

| 安全目标 | 原始 ASIL | 分解方案 | 分解后 ASIL | 理由 |
|----------|----------|----------|------------|------|
| SG-001 | C | 硬件监控 + 软件监控 | B(A) + B(A) | 独立检测路径 |
| SG-003 | A | 通信监控 + 安全状态 | QM(A) + QM(A) | 简单监控逻辑 |
