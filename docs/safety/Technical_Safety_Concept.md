## 5. 安全接口定义

### 5.1 温度传感器接口

```c
typedef struct {
    uint16_t temperature;      // 温度值 (0.1°C)
    uint8_t  status;           // 状态：0=正常，1=故障
    uint32_t timestamp;        // 时间戳 (ms)
} TempSensor_t;

// 接口函数
TempSensor_t TempSensor_Read(void);
bool TempSensor_IsValid(TempSensor_t sensor);
```

### 5.2 PWM 控制接口

```c
typedef struct {
    uint16_t duty_cycle;       // 占空比 (0-100%)
    uint16_t frequency;        // 频率 (Hz)
    bool     enabled;          // 使能状态
} PWMControl_t;

// 接口函数
void PWM_SetDuty(uint16_t duty);
void PWM_Enable(bool enable);
void PWM_EmergencyStop(void);  // 紧急停止
```

### 5.3 安全状态接口

```c
typedef enum {
    SAFE_STATE_NORMAL = 0,   // 正常状态
    SAFE_STATE_WARNING,      // 警告状态
    SAFE_STATE_FAULT,        // 故障状态
    SAFE_STATE_CRITICAL      // 紧急状态
} SafeState_t;

// 接口函数
SafeState_t SafetyManager_GetState(void);
void SafetyManager_Transition(SafeState_t newState);
```

## 6. 安全机制实现

### 6.1 端到端保护 (E2E)

**CAN 通信 E2E**:
```
发送端:
- 添加滚动计数器
- 添加 CRC 校验
- 添加数据标识

接收端:
- 验证滚动计数器
- 验证 CRC 校验
- 验证数据标识
```

### 6.2 程序流监控

**看门狗策略**:
```
主看门狗:
- 超时时间：100ms
- 喂狗周期：50ms
- 来源：主循环

子看门狗:
- 超时时间：20ms
- 喂狗周期：10ms
- 来源：关键任务
```

### 6.3 数据存储保护

**EEPROM 保护**:
```
- 使用 CRC32 校验
- 三重备份机制
- 写入前验证
- 读取后校验
```

## 7. 故障检测和响应

### 7.1 故障检测矩阵

| 故障类型 | 检测方式 | 检测时间 | 响应 |
|----------|----------|----------|------|
| 温度传感器开路 | 电阻检测 | <10ms | 关闭 PWM |
| 温度传感器短路 | 电压检测 | <10ms | 关闭 PWM |
| PWM 短路 | 电流检测 | <50ms | 关闭 PWM |
| CAN 总线错误 | 错误计数 | <100ms | 安全状态 |
| MCU 故障 | 看门狗 | <100ms | 复位 |

### 7.2 故障响应时间

| 故障等级 | 最大响应时间 | 安全动作 |
|----------|--------------|----------|
| 紧急 (Critical) | <1ms | 硬件切断 |
| 严重 (Severe) | <10ms | 软件关闭 |
| 一般 (Normal) | <100ms | 降级运行 |
| 轻微 (Minor) | <1s | 记录日志 |
