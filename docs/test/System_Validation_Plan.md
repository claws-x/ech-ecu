## 2. 测试策略

### 2.1 测试级别
系统确认测试在系统集成完成后执行，验证系统是否满足用户需求。

### 2.2 测试类型
| 测试类型 | 描述 | 方法 |
|----------|------|------|
| 功能测试 | 验证系统功能是否符合需求 | 黑盒测试 |
| 性能测试 | 验证系统性能指标 | 测量/监控 |
| 环境测试 | 验证系统在不同环境条件下的行为 | 环境模拟 |
| 安全测试 | 验证安全保护功能 | 故障注入 |

### 2.3 测试环境
- **硬件:** ECH ECU 原型板 + 负载模拟器
- **软件:** 测试脚本 + 数据采集工具
- **环境:** 温度/湿度可控测试室

## 3. 测试用例

### 3.1 温度控制功能测试

| 用例 ID | SYS-VAL-001 |
|---------|-------------|
| 测试目标 | 验证温度控制精度 |
| 前置条件 | ECU 上电，水温传感器连接正常 |
| 测试步骤 | 1. 设置目标温度为 60°C<br>2. 启动加热<br>3. 等待温度稳定<br>4. 记录实际温度 |
| 期望结果 | 实际温度在 60°C ±2°C 范围内 |
| 覆盖需求 | SYS-REQ-TEMP-001 |
| 优先级 | 高 |

| 用例 ID | SYS-VAL-002 |
|---------|-------------|
| 测试目标 | 验证温度过冲保护 |
| 前置条件 | ECU 处于加热状态 |
| 测试步骤 | 1. 设置目标温度为 50°C<br>2. 快速加热至 55°C<br>3. 观察功率调节 |
| 期望结果 | 功率自动降低，温度不超过 52°C |
| 覆盖需求 | SYS-REQ-TEMP-002 |
| 优先级 | 高 |

## 4. 验收标准

### 4.1 通过标准
- 所有高优先级测试用例 100% 通过
- 中优先级测试用例 ≥90% 通过
- 无严重缺陷 (Critical/Major)

### 4.2 退出标准
- 所有计划的测试用例执行完成
- 发现的缺陷已修复或记录
- 测试报告已生成并评审
