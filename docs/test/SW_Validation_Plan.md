## 2. 测试策略

### 2.1 测试级别
软件确认测试在软件集成完成后执行，验证软件是否满足软件需求和用户需求。

### 2.2 测试类型
| 测试类型 | 描述 | 方法 |
|----------|------|------|
| 功能测试 | 验证软件功能是否符合需求 | 黑盒/灰盒测试 |
| 接口测试 | 验证模块间接口正确性 | 集成测试 |
| 边界测试 | 验证边界条件处理 | 等价类/边界值 |
| 异常测试 | 验证异常情况处理 | 故障注入 |

### 2.3 测试环境
- **硬件:** ECH ECU 原型板 + 测试夹具
- **软件:** 单元测试框架 (Unity/CMock) + 覆盖率工具
- **配置:** 测试桩/模拟器
