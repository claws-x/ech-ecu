## 1. 系统架构概述

### 1.1 架构目标

本系统架构设计基于系统需求规格 (SyRS)，定义 ECH 水加热器 ECU 的整体架构，包括：
- 系统组件分解与接口定义
- 硬件与软件职责划分
- 系统级状态机与行为模型
- 与车辆其他系统的集成接口

### 1.2 设计约束

| 约束类型 | 描述 |
|----------|------|
| 硬件平台 | MCU: RH850 / AURIX（待定），12V 车载电源 |
| 工作环境 | -40°C ~ +85°C，IP67 防护 |
| 通信协议 | LIN 2.1 (19.2kbps) |
| 功能安全 | ASIL B (ISO 26262) |
| 诊断协议 | UDS (ISO 14229-1) over LIN |

## 7. 安全机制

### 7.1 安全需求分配

| 安全需求 | 实现机制 | 负责组件 | 验证方法 |
|----------|----------|----------|----------|
| SYS-S-001 (过温保护) | 温度阈值比较 + 硬件切断 | 温度控制 + PWM 驱动 | 故障注入测试 |
| SYS-S-002 (过流保护) | ADC 电流采样 + 快速响应 | ADC 驱动 + 状态管理 | 阶跃响应测试 |
| SYS-S-003 (通信丢失) | 超时计数器 + 安全状态 | 通信处理 | 总线关闭测试 |
| SYS-S-004 (传感器故障) | 范围检查 + 冗余比较 | ADC 驱动 | 开路/短路测试 |
| SYS-S-005 (看门狗) | 外部 WDG IC + 周期性喂狗 | 平台层 | 喂狗停止测试 |

### 7.2 故障检测与响应

| 故障类型 | 检测方法 | 响应时间 | 响应动作 |
|----------|----------|----------|----------|
| 过温 (≥85°C) | 温度阈值比较 | <10ms | 切断 PWM，进入故障状态 |
| 过流 (≥120%) | ADC 电流采样 | <10ms | 切断 PWM，进入故障状态 |
| CAN 丢失 | 超时计数器 | 500ms | 进入安全状态，发送故障码 |
| 传感器开路 | ADC 值超范围 | <50ms | 报错，降功率运行 |
| 传感器短路 | ADC 值超范围 | <50ms | 报错，降功率运行 |
| 看门狗超时 | 外部 WDG IC | 复位周期 | 系统复位 |

## 8. 系统集成策略

### 8.1 集成顺序

1. **阶段 1**: 硬件在环 (HIL) 基础验证
   - 电源、时钟、复位电路验证
   - JTAG/SWD 调试接口验证

2. **阶段 2**: 驱动层集成
   - PWM 驱动 + 示波器验证
   - ADC 驱动 + 信号源验证
   - CAN 驱动 + 总线分析仪验证

3. **阶段 3**: 服务层集成
   - PID 服务 + SIL 测试
   - 诊断服务 + UDS 测试工具

4. **阶段 4**: 应用层集成
   - 温度控制闭环测试
   - 状态机验证

5. **阶段 5**: 系统级集成
   - 实车 HIL 测试
   - 环境舱测试

### 8.2 集成检查点

| 检查点 ID | 检查项目 | 验收标准 | 责任方 |
|-----------|----------|----------|--------|
| SYS-IC-001 | 电源完整性 | 电压纹波 < 50mV | 硬件 |
| SYS-IC-002 | PWM 精度 | 频率误差 < 5%, 占空比误差 < 2% | 软件 + 硬件 |
| SYS-IC-003 | 温度精度 | 测量误差 < ±2°C | 软件 + 硬件 |
| SYS-IC-004 | CAN 通信 | 无丢帧，错误帧 < 0.1% | 软件 |
| SYS-IC-005 | 功能安全 | 所有安全机制响应时间达标 | 系统 |
