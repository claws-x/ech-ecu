## 1. 引言

### 1.1 目的

本配置管理计划定义 ECH 水加热器 ECU 项目的配置管理策略，确保：
- 所有工作产品 (Work Products) 的完整性和可追溯性
- 变更受控且可追溯
- 基线 (Baseline) 管理清晰
- 支持 ASPICE CL2 符合性

### 1.2 范围

本计划适用于 ECH ECU 项目的所有工作产品，包括：
- 需求文档 (SYS.1, SWE.1)
- 设计文档 (SYS.2, SWE.2/3)
- 源代码和测试代码
- 测试计划和报告 (SYS.4/5, SWE.4/5/6)
- 过程文档 (MAN.3, SUP.1/8/9/10)

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 基线 (Baseline) | 经正式评审和批准的工作产品集合，作为进一步开发的基础 |
| 配置项 (CI) | 需要配置管理控制的工作产品 |
| 变更请求 (CR) | 对已基线化配置项的修改请求 |
| 版本 (Version) | 配置项的特定状态标识 |

## 4. 版本控制

### 4.1 版本命名规则

**文档版本**: `vMAJOR.MINOR`
- MAJOR: 重大变更 (架构变更、需求变更)
- MINOR: 小幅修改 (勘误、澄清)

**代码版本**: `vX.Y.Z` (SemVer)
- X: 主版本号 (不兼容 API 变更)
- Y: 次版本号 (向后兼容功能)
- Z: 修订号 (向后兼容修复)

### 4.2 Git 分支策略

```
main (受保护，仅接受 Merge Request)
  │
  ├── develop (集成分支)
  │     │
  │     ├── feature/ech-pid-service
  │     ├── feature/ech-adc-driver
  │     └── feature/ech-can-driver
  │
  ├── release/v0.1 (发布分支)
  │
  └── hotfix/* (紧急修复)
```

### 4.3 提交信息规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**type 类型**:
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档变更
- `style`: 格式 (不影响代码运行)
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具链

**示例**:
```
feat(temp-ctrl): 添加 PID 温度控制算法

- 实现 PID 计算核心
- 添加参数标定接口
- 添加单元测试

Refs: SWE-F-004, SYS-F-004
```

## 5. 基线管理

### 5.1 基线定义

| 基线 ID | 基线名称 | 包含配置项 | 建立条件 | 用途 |
|---------|----------|-----------|----------|------|
| BL-001 | 需求基线 | CI-001, CI-002, CI-004 | 需求评审通过 | 软件设计基础 |
| BL-002 | 设计基线 | CI-002, CI-003 | 设计评审通过 | 代码实现基础 |
| BL-003 | 代码基线 v0.1 | 所有 src/ 代码 | 代码审查通过 | 测试基础 |
| BL-004 | 测试基线 v0.1 | 测试计划 + 报告 | 测试评审通过 | 发布候选 |
| BL-005 | 发布基线 v1.0 | 全部工作产品 | 项目评审通过 | 量产发布 |

### 5.2 基线发布流程

```
1. 准备阶段
   └─ CMO 确认所有 CI 已完成评审
   
2. 评审阶段
   └─ PM/TL/QA 进行正式评审
   
3. 批准阶段
   └─ PM 批准发布
   
4. 发布阶段
   └─ CMO 创建 Git Tag + 归档
   
5. 通知阶段
   └─ CMO 通知所有干系人
```

### 5.3 基线变更

基线发布后，任何变更需通过变更控制流程 (见第 6 章)。

                                             │
                    ┌────────────────────────┘
                    │
           ┌────────▼────────┐
           │   批准？        │
           └────────┬────────┘
                    │

         │                     │
