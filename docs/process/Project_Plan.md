## 1. 项目概述

### 1.1 项目背景

ECH (Electric Cabin Heater) 水加热器 ECU 是用于电动汽车座舱加热系统的控制单元。本项目旨在开发符合 ASPICE CL2 要求的 ECU 控制系统，包括需求分析、架构设计、软件实现、测试验证和过程文档。

### 1.2 项目目标

| 目标类型 | 目标描述 | 成功标准 |
|----------|----------|----------|
| **技术目标** | 开发符合车规要求的 ECU 控制系统 | 通过 ASPICE CL2 评估 |
| **质量目标** | 满足 ASPICE CL2 过程要求 | 过程符合性 ≥ 95% |
| **进度目标** | 按计划完成各里程碑 | 里程碑按时完成率 ≥ 90% |
| **成本目标** | 控制在预算范围内 | 实际成本/预算 ≤ 110% |

### 1.3 项目范围

**包含内容**:
- 系统需求分析与架构设计
- 软件需求分析与架构设计
- 软件详细设计与单元实现
- 单元测试、集成测试、系统测试
- ASPICE CL2 过程文档
- 配置管理与质量保证

**不包含内容**:
- 硬件设计与制造
- 生产工装开发
- 车规认证测试（由客户负责）

### 1.4 项目约束

| 约束类型 | 约束描述 | 影响 |
|----------|----------|------|
| **时间约束** | 2026-06-30 前完成 v1.0 发布 | 开发周期约 4 个月 |
| **技术约束** | 目标平台 RH850/AURIX，符合 ASIL B | 需使用车规级工具和流程 |
| **资源约束** | 团队规模 5-7 人 | 需合理分配任务 |
| **过程约束** | 符合 ASPICE CL2 要求 | 需执行完整的过程活动 |

## 12. 度量与分析

### 12.1 度量指标

| 指标类别 | 指标 | 公式 | 目标值 | 频率 |
|----------|------|------|--------|------|
| **进度** | 进度偏差 | (实际进度 - 计划进度)/计划进度 | ≤ 5% | 每周 |
| **成本** | 成本偏差 | (实际成本 - 预算成本)/预算成本 | ≤ 10% | 每月 |
| **质量** | 缺陷密度 | 缺陷数/代码行数 (KLOC) | ≤ 0.5 | 每周 |
| **质量** | 测试通过率 | 通过用例数/总用例数 | ≥ 95% | 每次测试 |
| **过程** | 过程符合性 | 符合过程活动数/总活动数 | ≥ 95% | 每月 |
| **资源** | 资源利用率 | 实际工时/计划工时 | 80-100% | 每周 |
